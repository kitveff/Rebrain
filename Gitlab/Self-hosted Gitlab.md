## Установка Self-Hosted GitLab

```
Обновим систему и установим необходимые зависимости:
 sudo apt-get update
 sudo apt-get install -y curl openssh-server ca-certificates tzdata perl
Для отправки электронных писем нам понадобится Postfix:
 sudo apt-get install -y postfix
При установке Postfix может появиться окно конфигурации.
Выберите Internet Site. В качестве mail name используйте внешнее DNS-имя, по которому доступна машина. 
Все остальные запрашиваемые параметры (если они возникнут) можно оставить по умолчанию.

Добавляем GitLab-пакет через запуск скачиваемого скрипта:
curl https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash

Запустите установку пакета, обязательно указав корректный EXTERNAL_URL. Это адрес, по которому GitLab будет доступен. В качестве этого адреса может выступать ip адрес ВМ или ее доменное имя. Если вы используете протокол https://, то будет автоматически сгенерирован сертификат от Let's Encrypt. Если установка является тестовой и сертификат не требуется, можно использовать http://. Для примера мы будем использовать доменное имя example.com с поддержкой TLS.
sudo EXTERNAL_URL="https://51.250.94.221:443" apt-get install gitlab-ce

После длительной установки с большим количеством логов процесс успешно завершится, и вы вернётесь к стандартной командной строке с сообщением о том, что GitLab успешно установлен. Для дополнительной проверки корректности установки платформы и первым знакомством с gitlab-ctl можно запустить команду проверки статуса и убедиться, что все компоненты находятся в состоянии run
sudo gitlab-ctl status

Если вы указали неверный адрес сервера GitLab, вы можете изменить его конфигурацию, отредактировав параметр external_url в файле /etc/gitlab/gitlab.rb:
   ##! For more details on configuring external_url see:
   ##! https://docs.gitlab.com/omnibus/settings/configuration.html#configuring-the-external-url-for-gitlab
   external_url 'https://example.com'

Мы снова воспользуемся утилитой gitlab-ctl. Нам нужно выполнить переконфигурацию, чтобы изменения (если они были внесены в предыдущем пункте) вступили в силу:
sudo gitlab-ctl reconfigure

По завершении процесса необходимо использовать автоматически сгенерированный пароль для авторизации в браузере. Пароль будет действительным в течение 24 часов:
cat /etc/gitlab/initial_root_password

На этом этапе вы можете отключиться от SSH, поскольку оставшаяся часть настройки будет проводиться в браузере. Чтобы начать, перейдите по адресу, указанному в пункте 5. Вы должны увидеть экран приветствия. Эта страница должна отображаться через HTTPS с новым сертификатом для этого домена.
Чтобы войти, используйте имя пользователя root и пароль из предыдущего пункта. После входа вам будет доступна начальная страница, где будут отображаться доступные проекты.

Перед настройкой учётной записи необходимо заполнить все необходимые поля. Однако, прежде чем начать, узнаем подробнее о конфигурации нашей установки. Для этого нажмите на иконку «бургер» в меню и выберите значок «гаечного ключа» с подписью Admin.
Вы попадёте на дашборд администратора, где можно узнать детальную информацию по пользователям, доступному функционалу в нашем GitLab-сетапе, а также список и версии всех компонентов и прочую статистику. Более подробно мы рассмотрим страницу администрирования в следующем блоке практикума
```
